# Design Notes

## Greeting node add-on JSON output safety

Issue reference: `tacogips/rielflow-packages`, title `Self-review greeting executable node-addon package JSON output safety`. No GitHub issue URL or issue number was provided; Step 1 intake uses the runtime title and requested behavior as the source of truth.

### Behavior

`packages/greeting-node-addon/addons/examples/greeting-shell/1/greeting.bash` must write a single mailbox payload at `$RIEL_MAILBOX_DIR/outbox/output.json`. The payload shape remains:

```json
{
  "greeting": "Hello <name>",
  "greetingIndex": "<index>",
  "timezone": "<timezone>",
  "timestamp": "<timestamp>"
}
```

The `name` argument, `GREETING_INDEX`, and `GREETING_TIMEZONE` are arbitrary strings for JSON-output purposes. Quotes, backslashes, tabs, and newlines in those values must preserve their string content and must not break JSON syntax. `timestamp` remains generated by `date` using the requested timezone behavior already present in the script.

### Boundaries

This issue is scoped to `packages/greeting-node-addon` and package-level metadata affected by that package's changed files. Do not modify unrelated packages or registry entries. README changes are needed only if the self-review finds install or executable grant usage is materially unclear.

The Bash script must not construct JSON with direct `printf` interpolation. It should delegate JSON encoding to available portable tooling and fail loudly before publishing malformed output if that tooling is unavailable or if timestamp generation fails.

If the implementation introduces an additional runtime dependency for JSON encoding, reflect it in `addons/examples/greeting-shell/1/addon.json` execution runtime hints and in `packages/greeting-node-addon/rielflow-package.json` add-on metadata so install-time inspection remains accurate.

### Metadata and Validation

Any change under `packages/greeting-node-addon/addons/examples/greeting-shell/1` requires refreshing the add-on `contentDigest` in `packages/greeting-node-addon/rielflow-package.json`. Any changed package payload or package metadata also requires package `checksum` and `integrity.digest` to be current.

Use the maintainer package-release guidance for verification. The generic `update-package-digests.ts` script skips `kind: "node-addon"` packages, so node-add-on digest correctness must be checked through package digest validation, expected to include:

```bash
task package:check-digests
```

Focused verification for this issue must also include an executable add-on smoke that runs `greeting.bash` with strings containing quotes, backslashes, and newlines, then parses `$RIEL_MAILBOX_DIR/outbox/output.json` and verifies the values are preserved.

### Review Feedback Addressed

Step 1 identified a high-severity finding: `greeting.bash` uses `printf` JSON interpolation and can emit invalid JSON for quoted, backslashed, or multiline inputs. This design requires JSON encoder-backed output and focused smoke coverage for those exact inputs.
